# <a name="quantum-oracles"></a><span data-ttu-id="d0e78-180">Quantum Oracle</span><span class="sxs-lookup"><span data-stu-id="d0e78-180">Quantum Oracles</span></span>

<span data-ttu-id="d0e78-181">Een Oracle $ O $ is een "Black Box"-bewerking die als invoer wordt gebruikt voor een ander algoritme.</span><span class="sxs-lookup"><span data-stu-id="d0e78-181">An oracle $O$ is a "black box" operation that is used as input to another algorithm.</span></span>
<span data-ttu-id="d0e78-182">Dergelijke bewerkingen worden vaak gedefinieerd met behulp van een klassieke functie $ f: \\ { 0, 1 \\ } ^ n \to \\ { 0, 1 \\ } ^ m, $ waarbij een $ n $ -bits binaire invoer wordt gebruikt en een $ m $ -bits binaire uitvoer wordt gegenereerd.</span><span class="sxs-lookup"><span data-stu-id="d0e78-182">Often, such operations are defined using a classical function $f : \\{0, 1\\}^n \to \\{0, 1\\}^m$ which takes an $n$-bit binary input and produces an $m$-bit binary output.</span></span>
<span data-ttu-id="d0e78-183">Als u dit wilt doen, moet u een bepaalde binaire invoer $ x = (x_ { 0 } , x_ { 1 } , \dots, x_ { n-1 } ) overwegen $ .</span><span class="sxs-lookup"><span data-stu-id="d0e78-183">To do so, consider a particular binary input $x = (x_{0}, x_{1}, \dots, x_{n-1})$.</span></span>
<span data-ttu-id="d0e78-184">We kunnen Qubit Staten labelen als $ \ket { \vec { x } } = \ket { x_ { 0 } } \otimes \ket { x_ { 1 } } \otimes \cdots \otimes \ket { x_ { n-1 } } $ .</span><span class="sxs-lookup"><span data-stu-id="d0e78-184">We can label qubit states as $\ket{\vec{x}} = \ket{x_{0}} \otimes \ket{x_{1}} \otimes \cdots \otimes \ket{x_{n-1}}$.</span></span>

<span data-ttu-id="d0e78-185">We kunnen eerst proberen om o te definiëren, $ $ zodat de $ o \ket { x } = \ket { f (x) } $ , maar dit heeft een aantal problemen.</span><span class="sxs-lookup"><span data-stu-id="d0e78-185">We may first attempt to define $O$ so that $O\ket{x} = \ket{f(x)}$, but this has a couple problems.</span></span>
<span data-ttu-id="d0e78-186">Ten eerste $ $ kan f een andere grootte hebben van de invoer en uitvoer ( $ n \ne m $ ), zodat $ $ het aantal qubits in het REGI ster kan worden gewijzigd.</span><span class="sxs-lookup"><span data-stu-id="d0e78-186">First, $f$ may have a different size of input and output ($n \ne m$), such that applying $O$ would change the number of qubits in the register.</span></span>
<span data-ttu-id="d0e78-187">Ten tweede, zelfs als $ n = m $ , mag de functie niet omkeerbaar zijn: als $ f (x) = f (y) $ voor sommige $ x \ne y $ , dan $ o \ket { x } = o y, \ket { } $ maar $ o ^ \dagger o \ket { x } \ne o ^ \dagger o \ket { y } $ .</span><span class="sxs-lookup"><span data-stu-id="d0e78-187">Second, even if $n = m$, the function may not be invertible: if $f(x) = f(y)$ for some $x \ne y$, then $O\ket{x} = O\ket{y}$ but $O^\dagger O\ket{x} \ne O^\dagger O\ket{y}$.</span></span>
